1 Purpose

This specification defines how Structural Cube commits a deterministic corpus, records resolver evidence, compares structural policies, rejects disguised regressions, and binds the accepted result into portable certificates.

The governing question is:

`Can structural authority increase while legality, exact replay, strict rank descent, solved closure, reference isolation, zero fallback, deterministic identity, and controlled route economy remain preserved?`

The improvement law is:

`preserved integrity + stronger declared authority + measured route benefit -> accepted improvement`

The anti-disguise law is:

`more structural labels != more structural authority`

A resolver change is not an improvement merely because it:

- renames a fallback;
- changes action boundaries;
- hides a reference route behind structural terminology;
- improves an average while worsening many seeds;
- shortens moves by removing evidence boundaries;
- changes the corpus after seeing the result;
- changes the move-counting metric without declaration;
- changes the policy without binding its identity;
- passes browser verification while failing independent replay.

6 State and Rank Requirements

Every source state must pass legal-state validation.

The structural state is:

`C = (P_e, O_e, P_c, O_c)`

Legal invariants are:

`sum(O_e) mod 2 = 0`

`sum(O_c) mod 3 = 0`

`parity(P_e) = parity(P_c)`

The structural rank is:

`W(C) = 24*rho + 4*(D_e + D_c) + tau_e + tau_c + F + T`

Every accepted route must satisfy:

`W(C_(i+1)) < W(C_i)`

for every completed action.

The terminal requirements are:

`final_W = 0`

`final_facelets = SOLVED`

A route fails if either condition is false.

13 Distribution Metrics

Every corpus report must include:

- count;
- minimum;
- mean;
- median;
- p95;
- maximum.

Averages alone are insufficient.

A result with a lower mean and severe tail regression must be classified as a trade-off, not an unqualified improvement.

Values are sorted in ascending numeric order before median or percentile calculation.

For an odd sample count, the median is the central sorted value.

For an even sample count, the median is the arithmetic mean of the two central sorted values.

For a non-empty sample of size n, the declared p95 method is the nearest-rank method with zero-based indexing:

`p95_index = ceil(0.95*n) - 1`

`p95 = sorted_values[p95_index]`

The index must be clamped to:

`0 <= p95_index <= n - 1`

The same method must be used for route moves, action counts, and signed move deltas.
