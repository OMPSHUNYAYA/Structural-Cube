1 Executive Summary

Structural Cube is a browser-based 3 x 3 x 3 cube system that resolves the current legal state through explicit structure, deterministic verification, graph-governed target selection, strict structural-rank descent, and portable evidence.

The current primary route is:

`current state -> canonical cubie state -> observed graph -> admitted pure-kernel targets -> graph ranking -> economy lookahead -> exact compiled action -> strict W descent -> rebuild -> solved state`

The primary interactive authority is:

`GRAPH_GOVERNED_ECONOMY_PRIMARY_AUTHORITY`

Auto Resolve and With Hint use this authority.

The resolver does not read a current-state reference solution or current-state reference distance.

`reference_route_access = NONE`

`current_state_reference_distance_access = NONE`

`fallback_activation = NONE`

The state convention is:

`C = (P_e, O_e, P_c, O_c)`

Legal states satisfy:

`sum(O_e) mod 2 = 0`

`sum(O_c) mod 3 = 0`

`parity(P_e) = parity(P_c)`

The structural potential is:

`Phi(C) = (rho, D_e + D_c, tau_e + tau_c, F, T)`

The integer rank is:

`W(C) = 24*rho + 4*(D_e + D_c) + tau_e + tau_c + F + T`

For every legal state:

`0 <= W(C) <= 126`

and:

`W(C) = 0 iff C = SOLVED`

Every admitted action must satisfy:

`W(C_(i+1)) < W(C_i)`

The graph-governed economy policy is:

`full graph ranking -> top 4 admitted targets -> depth 4 structural lookahead -> lowest compiled face-turn cost`

The cost signal is computed only from exact realizations of isolated pure-kernel effects. It is not obtained from a solution route for the current cube.

5 Canonical State and Legality

The canonical cube state uses 54 facelets in this order:

`U, R, F, D, L, B`

Each face is read row by row, left to right, top to bottom.

The canonical solved state is:

`UUUUUUUUURRRRRRRRRFFFFFFFFFDDDDDDDDDLLLLLLLLLBBBBBBBBB`

Cubie arrays use:

`perm[position] = piece identity occupying that position`

`ori[position] = orientation value at that position`

A legal state must satisfy:

- exactly nine facelets of each colour;
- a legal centre orientation;
- unique edge and corner pieces;
- edge orientation parity;
- corner orientation parity;
- matching edge and corner permutation parity.

Invalid, incomplete, contradictory, or unreachable input must be refused.

`invalid state -> no structural route claim`


6 Structural Rank

6.1 Components

Let:

`rho = common edge and corner permutation parity flag`

`D_e = 12 - cycles(P_e)`

`D_c = 8 - cycles(P_c)`

`tau_e = number of edge 2-cycles`

`tau_c = number of corner 2-cycles`

`F = number of flipped edges`

`T = sum_i min(O_c[i], 3 - O_c[i])`

The lexicographic potential is:

`Phi(C) = (rho, D_e + D_c, tau_e + tau_c, F, T)`

The integer rank is:

`W(C) = 24*rho + 4*(D_e + D_c) + tau_e + tau_c + F + T`

6.2 Boundary

For every legal state:

`0 <= W(C) <= 126`

The solved-state identity is:

`W(C) = 0 iff C = SOLVED`

6.3 Action Law

A completed structural action must satisfy:

`W_after < W_before`

A route is accepted only when:

- every action boundary strictly descends;
- the complete route reaches `W = 0`;
- the resulting facelet state is solved;
- every exact action effect replays successfully;
- the action chain remains continuous.

The finite descent basis is:

`finite non-negative W + strict descent -> finite action count`

The declared maximum is:

`action_count <= W(C_0) <= 126`


7 Pure Structural Kernels

The declared pure-kernel family is:

`EF(i,j)`

`CT(i,j)`

`E3(i,j,k)`

`C3(i,j,k)`

`PB(i,j;k,l)`

Meaning:

- `EF` changes only the declared edge-flip residue;
- `CT` changes only the declared corner-twist residue;
- `E3` applies one declared edge 3-cycle;
- `C3` applies one declared corner 3-cycle;
- `PB` applies one edge swap and one corner swap together.

The deterministic case order is:

1. common odd parity -> `PB`;
2. a permutation cycle of length at least three -> `E3` or `C3`;
3. paired 2-cycles -> deterministic 3-cycle repair;
4. identity permutations with edge-flip residue -> `EF`;
5. corner-twist residue -> `CT`.

Each target contract binds:

- kernel type;
- piece identities;
- direction or power where applicable;
- predicted legal state;
- predicted rank;
- canonical contract text;
- contract SHA-256.


8 Constructive Kernel Compiler

The compiler constructs an exact face-turn realization of an isolated pure-kernel effect.

The compiler authority boundary is:

`compiler_search_input = ISOLATED_PURE_KERNEL_EFFECT_ONLY`

The compiler does not solve the current cube.

It receives a pure-effect target, constructs a deterministic realization, and returns:

- normalized move sequence;
- exact target effect;
- compiler profile;
- compiler hash;
- repeatability evidence.

Admission requires:

`compiled sequence applied to current state = predicted complete cubie state`

and:

`W_after < W_before`

A realization that fails exact replay is refused.

19 Refusal and Failure Behaviour

The application must refuse a route when:

- the state is illegal;
- a structural target cannot be constructed;
- a compiler realization cannot be verified;
- exact effect differs from the contract;
- W does not strictly decrease;
- chain continuity fails;
- final state is not solved;
- a hidden reference or fallback path is detected.

The rule is:

`unverified route -> no solved claim`

A refusal is preferable to an unsupported structural claim.

23 Deployment Direction

The next architecture work should preserve v1.0.2 behaviour while increasing evidence breadth.

Priority directions are:

1. Stabilize SCRANK, kernel-contract, compiler, graph-policy, and SCCERT specifications under independently reviewable canons.
2. Produce an independently written route producer that reaches the same declared decisions without sharing the browser implementation.
3. Extend testing beyond P100 with long-cycle, orientation-heavy, parity-boundary, and manually constructed legal corpora.
4. Add systematic mutation corpora for every certificate field and every root domain.
5. Improve compiled move economy without weakening graph authority or strict descent.
6. Measure learning outcomes separately from resolver correctness.
7. Generalize the architecture to other finite transformation systems only after their legal state, rank, action contracts, and evidence canons are explicit.

The governing deployment law is:

`broader claim -> broader independent evidence`
